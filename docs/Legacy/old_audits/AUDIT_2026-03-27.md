## Section G: What Could Still Be Wrong

1. **Trade 505466a5 stuck "open" in DB** — This 0DTE option expired worthless but the DB record was never closed. Every iteration's _check_exits calls `get_positions()` from the broker. If the option expired, the broker removes it from positions, so _check_exits never sees it and never updates the DB. There is no "stale trade cleanup" mechanism. This means $352 loss is not reflected in P&L.

2. **0DTE entry after EOD cutoff** — The trade entered at 15:46, 1 minute after the 15:45 scalp EOD cutoff. The entry pipeline has no equivalent time-of-day gate. The EOD exit fires on the NEXT iteration, but by then the order may not have filled. This is a race condition.

3. **TSLA swing Greeks unverified** — We fixed the EV input and threshold, but we have zero evidence that get_greeks() returns correct delta/gamma/theta for 7-45 DTE TSLA options. If the fallback delta estimator fires (because Lumibot returns delta≈0), the EV calculation uses synthetic Greeks that may not match reality.

4. **VIX regime adjustment on TSLA untested** — Step 5.5 applies VIX regime multipliers to swing predictions. We don't know the current VIX level or which regime multiplier was applied. If VIX is elevated (it should be, market was down), a 0.7x high-vol multiplier could reduce a 0.45 confidence signal to 0.315 — still above 0.22 but reducing EV input.

5. **Theta acceleration factor is aggressive** — For DTE 7-13, theta_accel = 1.5x. For DTE < 7, it's 2.0x. This is conservative (protects against underestimating theta) but it means the EV formula systematically underestimates the attractiveness of shorter-dated options. A 7-DTE ATM option that might be profitable in practice gets rejected by the formula.

6. **Calibrator lost on scalp incremental models** — The scalp incremental model was using `calibrator=None` (the isotonic calibrator from the base model was dropped). We patched this today. But prior trades made while the calibrator was missing may have used uncalibrated probabilities, affecting confidence values and sizing.

7. **No fill confirmation in entry pipeline** — The code calls `submit_order()` and immediately logs the trade to DB with the estimated premium. If the order doesn't fill (e.g., market moving, no liquidity), the DB has a "trade" that never actually happened. The `on_filled_order` callback updates the price but there's no mechanism to cancel a trade record if the order is rejected or expires unfilled.

8. **Model health monitoring has no downstream effect** — The `_prediction_history` tracking computes accuracy metrics but never actually pauses trading when the model degrades. It's informational only.

9. **signal_log CSV was multi-day** — The export included March 25 and 26 data, which inflated kill counts. The 292 "equity gate" kills were actually from March 25 when the portfolio was $5K (below OTM's $25K requirement). Automated analysis of signal logs should always filter by date.

10. **underlying_reversal_pct on SPY scalp is 1.0%** — For SPY (lower volatility than TSLA), 1.0% = ~$6.34. On a day like today with SPY down 1.5%, a 1% adverse move is very common. If a scalp position is held for more than a few minutes, the underlying reversal exit could fire prematurely. For scalp positions with max_hold_days=1, the underlying reversal may be redundant with the stop loss.
